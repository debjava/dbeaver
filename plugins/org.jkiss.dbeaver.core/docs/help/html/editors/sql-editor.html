SQL Editor

Overview

SQL editor allows editing and execution of SQL queries and scripts.

There are two ways to open SQL editor - using "SQL Editor" or "New SQL Editor" commands. These commands are accessible through main toolbar or main menu or with shortcuts CTRL+L and CTRL+ALT+L.

"SQL Editor" command opens recent SQL script for currently active connection (active connection name shown in main toolbar) or creates new SQL script (which can be opened later from Project Explorer view). "New SQL Editor" always creates new script.

When you close SQL editor it will be auto-saved. If you want to revert changes - use appropriate command from context menu.

SQL editor highlights SQL syntax which structure (keywords) depends on current database SQL dialect. When you open SQL editor for some connection DBeaver will try to connect to underlying database. If database is inaccessible then you won't be able to execute queries.

You may change active database of current SQL editor at any moment using main toolbar.

There are two SQL execution modes: query and script.

Query execution

Query execution command accessible from toolbar, context menu, main menu or with shortcut CTRL+Enter.

SQL editor extract current query - it is SQL statement under cursor limited by empty lines or statement delimiter (";" symbol)

Also you may select any part of text - it will be executed as single query.

Query execution usually produces some result set - it will be printed in "Data Grid" tab. Read more about Result Sets navigation and edit.

Scripts execution

Script is a whole content of SQL editor or selected text. This command is accessible from menu, toolbar or with shortcut ALT+X.

SQL editor parses content using ";" character as delimiter between separate statements and then executes these statements consequently.

Execution plan

You may automatically build execution plan for current query. This command accessible from toolbar or with shortcut CTRL+SHIFT+Enter.

Driver provider must support execution plan extraction. Generic driver provider doesn't support this feature.

Execution log

Execution log tab contains history of all command executed in current SQL editor.
