Driver Editor

This dialog allows user to change any driver settings.

Basic settings

Driver nameName of driver, it'll be shown in driver manager. Also it could be used by DBeaver to name scripts, show tips, etc
CategoryOptional field. Some databases have many different drivers, it is convenient to group these drivers. Category is just a name for this group. Most drivers do not have category.
Class nameRequired field. It is a name of Java class which implements driver. You may find class name in driver's documentation or you may click on "Find Class" button in the "Driver Libraries" tab and choose class name from the list.
Sample URLOptional field. If it's value is left blank then user has to specify full database URL when he/she creates new database connection using this driver. It is much more convenient when sample URL is specified - in this case connection wizard will require only valuable connection properties (such as host name, database name, etc).

Sample URL is a JDBC database URL but instead of real database properties it consists from "patterns" which will be replaced with real values by connection wizard.

List of supported URL patterns:

  • {host} - database host name
  • {port} - database port number
  • {database} - database/schema name
  • {server} - database server name. Rarely used, usually by complex legacy databases (e.g. Informix)
  • {folder} - database folder. Usually used by embedded database engines to specify folder with database files
  • {file} - the same as {folder} but specifies single file

Optional patterns may be enclosed in square brackets

Typical sample URL: jdbc:db2://{host}[:{port}]/{database}

Driver Libraries

List of external driver libraries. Usually library is a jar or zip archive file which contains implementation of JDBC driver. Also You may specify entire directory as source of driver classes instead of archive file.

"Find class" button bellow library list searches for all JDBC driver implementations in specified libraries. Then you may choose particular class name (if there are multiple classes found) and it will be placed in "Class Name" field.

Connection properties

Default additional properties for driver. These properties will be used for all connections based on this driver. However these properties could be redefined in particular connection.

Additional properties are used by drivers to configure drivers for specific environment or for specific requirements. List of available properties is specific for each driver. You may find them in driver's documentation or in connection wizard ("Advanced properties" tab)

Advanced parameters

Advanced parameters are used by some driver provides to configure driver's behaviour.

List of parameters supported by Generic driver provider:

Shutdown parameter Some embedded database engines requires additional shutdown parameter in URL when you disconnect from database. Otherwise driver will lock database file until you finish DBeaver process. If specified then shutdown parameter will be appended to database URL on disconnect. Usually shutdown parameter looks like ;shutdown=true
Get active database query SQL query which returns name of current database/schema. Used by DBeaver to show active database/schema in UI.
Set active database query SQL query which changes current database/schema.

License

Driver license. If present then user should read and accept this license before use of this driver.
