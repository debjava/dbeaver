Всех с прошедшими праздниками!

По роду деятельности мне регулярно приходится работать с различными СУБД (как программисту, администратору или юзеру), за последние годы пришлось поработать практически со всеми разспространёнными и со многими узкоспециальными БД.
Мне, как и многим, всегда хотелось иметь один универсальный инструмент для работы с любыми базами. Помимо универсальности требуется кроссплатформенность и расширяемость с поддержкой особенностей конкретных СУБД. Ну и хотелось бы бесплатности.
На первый взгляд, на рынке масса разных продуктов однако всё не так хорошо как хотелось бы. Наиболее популярны два - SQL Squirrel и DB Visualizer. Все остальные либо недостаточно универсальны (работают с ограниченным набором СУБД) либо крайне неудобны в использовании. При этом в Squirrel кошмарный UI, кучи багов, плюс он многого не умеет. Бесплатная версия DBVis умеет ещё меньше, хотя и UI и качество безусловно лучше.
В общем, я провёл довольно много времени в поисках подходящего инструмента, пересмотрел несколько десятков разных и - не нашёл.

В результате, как-то само-собой вышло так, что весь последний год с небольшим моим weekend hobby было написание собственного универсального инструмента для работы с СУБД.
За прошедшее время он приобрёл достаточно товарный вид чтобы было не стыдно показать общественности. Итак - <a href="http://dbeaver.jkiss.org">DBeaver</a>.

Название DBeaver выбралось само собой - в пику SQuirreL. И, хочется верить, бобёр получился и симпатичней и работоспособней, чем белка.
Написано всё на Java с использованием SWT, т.е. UI на всех ОС (с оговорками для Mac OS) выглядит нативным.
На вопрос "почему Java" есть простой ответ - для подобной задачи на данный момент просто не существует альтернативы JDBC. А возможность работать из одного клиента с любой базой без установки нативных клиентов, регистрации в ODBC и т.п. - это главное требование.
По фичам: всё довольно стандартно.
Для любой БД есть как минимум
- Исполнение SQL запросов (с плюшками редактора типа автокомплита, гиперссылок на метаданные, форматирования и т.п.)
- Работа с resultset-ами (скроллирование, редактирование, ордеринг, фильтрация)
- Просмотр метаданных
- Работа с типами LOB
- Работа с транзакциями
- Поиск объектов БД по имени
- ERD диаграммы
- Экспорт данных из таблиц и произвольных выборок во внешние файлы (csv, sql).

Поддерживается система плагинов и для конкретной СУБД набор возможностей может быть расширен. В первой версии существует только плагин для MySQL.
На данный момент он умеет не очень много, но лучше чем ничего -
- Работа с сессиями
- Работа с юзерами и правами доступа
- Создание/удаление схем
- Просмотр процедур/триггеров и DDL для таблиц
- Поддержка типов данных enum и set.

DBeaver был оттестирован на MySQL, PostgreSQL, Oracle, SQL Server, DB2, Derby/JavaDB, HTSQL, ODBC и некоторых экзотических СУБД.
Наверняка хватает и багов - т.к. мне пока остро не хватает QA.

В ближайших планах доработка плагина MySQL - добавление редактора метаданных (таблиц и т.д.) и редактора/отладчика процедур и триггеров.
Далее плагин для Oracle и проч. Параллельно отладка/улучшение существующего функционала и доработка сайта (как минимум форум).

От этого поста помимо какого-никакого пиара конечно хотелось бы фидбэка с максимумом критики. Активных известных мне пользователей DBeaver на сегодня всего пара десятков человек.
После минимального распространения с помощью download сайтов какие-то люди заходят и скачивают, но фидбэка от них никакого, конечно.
